### 2.2 重映射目标

原始 D-Grasp 使用的是 MANO 高维手模型，不能直接用于当前机器人。现在改成了面向 FR5 + RH56E2 的中间数据格式:

- 机械臂: 不再直接使用目标 6 轴关节角

- 灵巧手控制目标: `6D` 手指关节角
- 关键点目标: 从原始 `21` 个 MANO 关键点压缩成 `7` 个语义点
- 接触标签: 从原始 `16` 维链节接触重映射成 `12` 维机器人接触标签

## 4. 当前奖励设计

当前奖励是“位姿对齐 + 接触匹配 + 稳定性约束”的混合设计。

配置位置:

- [default_task.json](E:/A_mujoco/fr5_rh56e2_dgrasp_rl/config/default_task.json)

实现位置:

- [env.py](E:/A_mujoco/fr5_rh56e2_dgrasp_rl/fr5_rh56e2_dgrasp_rl/env.py)

奖励项及权重:

- `site_reward = 2.0`
  - 7 个语义点相对物体的位置误差
- `pose_reward = 0.1`
  - RH56E2 `6D` 关节目标误差
- `wrist_pose_reward = 1.0`
  - wrist 相对物体的平移误差
- `wrist_align_reward = 0.5`
  - wrist 相对物体的旋转误差
- `contact_reward = 1.0`
  - `12D` 接触位逐位匹配
- `impulse_reward = 2.0`
  - 目标接触位的真实接触力
- `falling_reward = 1.0`
  - 抓取后物体滑落惩罚
- `rel_obj_reward = -1.0`
  - 物体线速度/角速度惩罚
- `body_vel_reward = -0.5`
  - 手掌线速度惩罚
- `body_qvel_reward = -0.5`
  - 手掌角速度惩罚
- `penetration_reward = -0.75`
  - 手物穿透惩罚
- `action_rate = -0.01`
  - 动作抖动惩罚

总 reward 还会做下限裁剪:

- `reward_clip = -2.0`

当前设计原则:

- 不直接模仿 MANO 高维关节
- 优先对齐“机器人可执行的抓取语义”
- 接触奖励采用软匹配，而不是只看 `12/12 exact match`

## 8. 当前问题与后续方向

当前主要问题:

1. 数据重映射已经基本可用，但部分样本仍处于接触边界附近。
2. 训练 reward 能下降惩罚，但还没有形成真正抓取成功。
3. viewer 回放虽然支持动态模式，但抓取执行过程还不够自然。
4. 训练仍是 CPU，尚未利用本机 RTX 3060。

建议的下一步:

1. 基于当前 `18` 条样本继续保留全量训练，不退回只用 `7` 条。
2. 调整 reward，让“抓稳物体”比“少移动、少碰撞”更占优势。
3. 把训练时长从 `50` updates 提高到 `200~500` updates。
4. 后续视需要切换到 CUDA 版 PyTorch，尝试 GPU 加速训练。
